# Git Repository File Guide

This document explains every file and folder in the project, whether it should be committed to git, and why.

## Files to Commit (YES — add to git)

### Source Code

| File / Folder | Purpose |
|---|---|
| `mcp_context_server/` | **Main application code.** All 7 Python modules that make up the MCP server: entry point, config, sessions, Jira client, Confluence client, Git service, and knowledge base search. |
| `mcp_context_server/__init__.py` | Package marker — makes `mcp_context_server` a Python package. |
| `mcp_context_server/main.py` | MCP server entry point. Registers all tools via `@mcp.tool()` decorators. |
| `mcp_context_server/config.py` | Settings loaded from environment variables with `MCP_` prefix using pydantic-settings. |
| `mcp_context_server/sessions.py` | Session model and file-based JSON persistence for cross-conversation context. |
| `mcp_context_server/jira_client.py` | Async Jira REST API client using httpx. |
| `mcp_context_server/confluence_client.py` | Async Confluence REST API client using httpx. |
| `mcp_context_server/git_service.py` | Git operations (log, diff, blame, status) via GitPython. |
| `mcp_context_server/knowledge_base.py` | Filesystem search, file reading, grep with regex support. |

### Project Configuration

| File / Folder | Purpose |
|---|---|
| `pyproject.toml` | **Project definition file.** Declares package name, version, Python version requirement (>=3.11), all runtime and dev dependencies, CLI entry point, linter config, and test config. This is how `pip install` knows what to install. |
| `.env.example` | **Template configuration file.** Shows all available environment variables with documentation. Users copy this to `.env` and fill in their values. |
| `.gitignore` | Tells git which files/folders to exclude from version control. |
| `LICENSE` | MIT license file. |
| `README.md` | Project documentation — features, install instructions, configuration reference, architecture overview. |

## Files to EXCLUDE (NO — do NOT add to git)

### `.venv/` — Python Virtual Environment

A local copy of the Python interpreter and all installed packages (~50-200 MB). It contains OS-specific compiled binaries and is recreated by anyone cloning the repo via:
```bash
python -m venv .venv
pip install -e ".[dev]"
```
**Excluded by `.gitignore`.**

### `.pytest_cache/` — Pytest Cache

Auto-generated cache created by pytest to speed up re-runs (tracks last-failed tests, etc.). Recreated automatically when running `pytest`. **Excluded by `.gitignore`.**

### `mcp_context_server.egg-info/` — Package Metadata

Auto-generated by `pip install -e .` (editable install). Contains derived metadata like version, dependencies, and entry points from `pyproject.toml`. Recreated automatically on install. **Excluded by `.gitignore` via `*.egg-info/`.**

### `__pycache__/` — Python Bytecode Cache

Auto-generated `.pyc` compiled bytecode files created by Python at runtime. Exists in both `mcp_context_server/` and `tests/`. Recreated automatically. **Excluded by `.gitignore`.**

### `.env` — Local Environment Configuration

Contains **your personal** Jira/Confluence credentials, API tokens, and local filesystem paths. Must never be committed to git for security reasons. Each user creates their own from `.env.example`. **Excluded by `.gitignore`.**
